Linux系统安装jdk、Tomcat、Mysql、Git、Maven

发表于2022-10-27|更新于2022-11-03
|阅读量:

一、安装方式

1、二进制发布包安装：软件已经针对具体凭条编译打包发布，只要解压，修改配置即可
2、rpm安装：软件已经按照redaht的包管理规范进行打包，使用rpm命令进行安装，不能自行解决依赖库问题
3、yum安装：一种在线软件安装方式，本质上还是rpm安装，自动下载安装包并安装安装过程中自动解决库依赖问题
4、源码编译安装：软件以源码工程的形式发布，需要自己编译打包

1、安装JDK（二进制发布包）步骤：

1）使用FinalShell将压缩包上传到Linux系统
2）使用tar命令解压 tar -zxvf jdk-8u171-linux-x64.tar.gz -C /usr/local
3）配置环境变量&Path JAVA_
HOME=/usr/local/jdk1.8.0_171
PATH=$JAVA_HOME/bin:$PATH
4）source /etc/profile 让文件生效 java -version

2、安装Tomcat（二进制发布包）步骤：

1）tar -zxvf apache-tomcat-7.0.57.tar.gz -C /usr/local
2）cd /usr/local/apache-tomcat-7.0.57/bin
3）sh startup.sh
验证Tomcat是否启动成功：
1）查看启动日志
more /usr/local/apache-tomcat-7.0.57/logs/catalina.out
或者
tail -50 /usr/local/apache-tomcat-7.0.57/logs/catalina.out
2）查看系统进程
ps -ef|grep tomcat （ps -ef可以查看 当前运行的所有进程信息）

启动成功后要关闭防火墙才可以在外网访问到 —-> centosip地址:8080
防火墙操作命令：
一.系统防火墙
1.启动防火墙 systemctl start firewalld
2.关闭防火墙 systemctl stop firewalld
3.查看状态 systemctl status firewalld
4.开机启用防火墙 systemctl enable firewalld
5.开机禁用防火墙 systemctl disable firewalld
二.某个端口的防火墙
1.开放某个端口，如8080端口 firewall-cmd –zone=public –add-port=8080/tcp –permanent
重新加载配置 firewall-cmd –reload
2.查看防火墙锁开放的端口 firewall-cmd –zone=public –list-ports
3.查看某个端口的访问权限，如8080 firewall-cmd –zone=public –query-port=8080/tcp
4.关闭某个端口的防火墙，如8080端口 firewall-cmd –zone=public –remove-port=8080/tcp –permanent

但是为了系统的安全，我们一般是会开启防火墙的，不过只是开启对应端口以供外网访问

停止Tomcat服务：
1）bin下的 sh shutdown.sh 或者 ./shutdown.sh
2）结束Tomcat进程 ps -ef|grep tomcat查看进程id， kill -9 id
建议使用shutdown来结束服务

3、安装MySQL （RPM安装 Red-Hat Package Management）步骤（其实这种安装方法是很繁琐的，推荐使用docker）

1）CentOS7自带的mariadb跟MySQL冲突，需要先卸载 rpm -qa | grep mariadb
2）卸载命令：rpm -e –nodeps 软件名称
3）mkdir /usr/local/mysql tar -zxvf mysql-5.7.25-1.el7.x86_64.rpm-bundle.tar.gz -C /usr/local/mysql
4） 进入/usr/local/mysql文件夹，看到6个rpm软件包，要依次安装 `
rpm -ivh mysql-community-common-5.7.25-1.el7.x86_64.rpm
rpm -ivh mysql-community-libs-5.7.25-1.el7.x86_64.rpm
rpm -ivh mysql-community-devel-5.7.25-1.el7.x86_64.rpm
rpm -ivh mysql-community-libs-compat-5.7.25-1.el7.x86_64.rpm
rpm -ivh mysql-community-client-5.7.25-1.el7.x86_64.rpm
yum install net-tools
rpm -ivh mysql-community-server-5.7.25-1.el7.x86_64.rpm

启动MySQL
systemctl status mysqld 查看服务状态
systemctl start mysqld 启动mysql服务
开机启动
ps -ef | grep mysql
netstat -tunlp 查看已经启动的服务
netstat -tunlp | grep mysql

登录MySQL数据库，查阅临时密码，注意空格
cat /var/log/mysqld.log
cat /var/log/mysqld.log | grep password

登录mysql修改密码，开启访问权限
mysql -uroot -p 登录mysql（使用临时密码）

set global validate_password_length=4; 设置密码最低位数
set global validate_password_policy=LOW; 设置密码安全登记
set password =password(‘root’)

开启访问权限
grant all on . to ‘root‘@’%’ identified by ‘root’;
flush privileges;

外部链接虚拟机数据库记得开启3306端口号

安装lrzsz（要用于Linux文件的上传与下载）

1）yum list lrzsz （搜索lrzsz）
2）yum install + 包名
3）安装完成之后可以使用 rz命令

手动项目部署

1）在idea中将项目打成jar包
2）java -jar启动springboot程序
3）让程序在后台运行并将日志输出到日志文件
a.nohup命令 ： no hang up –> nohup java -jar boot工程名.jar &> hello.log &
后台运行程序并将日志输出到hello.log文件
b.停止对应 程序 则使用杀掉对应进程的方式：ps -ef|grep java kill -9 进程号

通过shell脚本自动部署项目

1）在Linux安装Git
yum list git 列出git所有的安装包
yum install git 安装git
2）在Linux安装Maven
tar -zxvf apache-maven-3.5.4-bin.tar.gz -C /usr/local
vim /etc/profile
export MAVEN_HOME=/usr/local/apache-maven-3.5.4
export PATH=$JAVA_HOME/bin:$MAVEN_HOME/bin:$PATH
PATH=$JAVA_HOME/bin:$PATH
source /etc/profile
mkdir repo 本地仓库
cd apache-maven-3.5.4
cd conf
vim settings.xml
加入/usr/local
3）编写Shell脚本（拉去代码、编译、打包、启动）
4）为用户授予执行Shell脚本的权限
chmod 777 bootStart.sh 为所有用户授予读、写、执行权限
chmod 755 bootStart.sh 为文件拥有者授予读、写、执行，同组用户和其他用户 读、执行
chmod 210 bootStart.sh 为文件拥有者授予写，同组执行，其他无权限
5）执行Shell脚本
